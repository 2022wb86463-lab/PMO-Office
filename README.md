# PMO-Office

## Project Overview
The PMO-Office repository is used to centrally manage project tracking assets for a distributed Project Management Office (PMO) team. It demonstrates basic Git and GitHub operations such as repository creation, cloning, committing, and pushing changes.

## Repository Contents

- `README.md` – Project overview and repository information.
- `Inventory.md` – Inventory of project documents.
- `Project_Plan.txt` – Initial project planning document.

## Objectives

- Create a centralized cloud repository.
- Track project documentation.
- Collaborate using Git and GitHub.
- Synchronize local and remote repositories.

## Git Workflow Used

1. Create a GitHub repository.
2. Add an initial inventory file.
3. Clone the repository locally.
4. Create and modify project files.
5. Stage and commit changes.
6. Push changes to GitHub.
